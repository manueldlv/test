Chapter 1

What is Vibe Coding?

Vibe Coding means building digital products by combining your design taste with AI-assisted coding tools. You do not need to memorize programming first. You learn by shipping.

Why designers should learn it

You can move from static mockups to real interactive experiences and test ideas faster.

What you can build

Portfolios, landing pages, marketing sites, design systems docs, and small SaaS interfaces.

What changes for your career

You become a designer who can prototype, collaborate with engineers, and ship independently.

Tip: Think like a product designer. Start small, test quickly, and improve with each iteration.

Chapter 2

Programming, Local Servers, and Localhost

Let’s remove the mystery. These concepts are simple once explained in plain language.

01

What programming actually is

Programming is writing clear instructions so your computer can build and show your design decisions.

02

What a local server is

A local server runs your website on your own machine so you can preview it before publishing.

03

What localhost means

localhost is your computer talking to itself. It is your private testing address.

Key concept: You are not “breaking the internet” while learning. You are experimenting safely on your own machine.

Chapter 3

Basic Tools You Need

You only need a few tools to start: code editor, terminal, and browser.

What is a code editor?

It is your design canvas for code. Visual Studio Code is the easiest place to begin.

  • Install VS Code
  • Open a folder/project
  • Edit files and save changes

What is a terminal?

It is a text-based control panel where you run commands like “start project” or “save changes”.

Beginner terminal commands

pwd      # show current folder
ls       # list files
cd name  # enter folder
mkdir x  # create folder
code .   # open project in VS Code

Chapter 4

Working With Projects, Git, and GitHub

This is the collaboration layer. You and AI can code, but Git helps you manage versions safely.

Repository

Your project folder with full history.

Git

The version control engine on your machine.

GitHub

The cloud home for your repository.

Commit

A saved checkpoint with a short message.

Branch

A safe lane for testing changes.

Pull Request

A request to merge work into the main version.

git clone https://github.com/your-name/project-name.git
cd project-name
git checkout -b improve-homepage
git add .
git commit -m "Improve hero layout"
git push origin improve-homepage

Warning: Never commit random files. Always check what changed before running git add .

Chapter 5

Working With AI as a Design Partner

AI helps you write, explain, and debug code. Your role is directing it with clear context.

What Vibe Coding means in practice

Describe the visual outcome, constraints, and expected behavior. Ask AI to generate the smallest useful step.

Prompt formula for designers

Goal + Context + Constraints + Output format = higher-quality code and fewer rewrites.

I am a beginner designer.
Goal: Create a pricing section.
Context: Existing site uses Inter and a clean white layout.
Constraints: Vanilla HTML/CSS/JS only.
Output: Return full code with comments and explain each block simply.

Chapter 6

The Real Workflow You Will Repeat

This is your practical weekly loop as a designer who codes.

  1. Clone a project from GitHub
  2. Open it in VS Code
  3. Run the development server
  4. Edit code and preview live
  5. Make a commit
  6. Push to GitHub
  7. Create a pull request
A

Run local server

Use npm run dev and open the shown localhost URL.

B

Edit and save

Each save updates the preview. This is your fastest design feedback loop.

Chapter 8

Best Practices for Confident Progress

You do not need to be perfect. You need a safe process.

01

How to read errors

Read the first error line first. It usually points to the exact file and line to fix.

02

How to ask AI for help

Share the error, expected behavior, and file snippet. Ask AI for minimal changes only.

03

How to avoid breaking the project

Work in branches, commit often, and test after each small change.

Final reminder: Coding is a design medium. The more you practice small steps, the faster your confidence grows.
